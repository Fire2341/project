# MEMORY.md - 调教记录

## 🚨 关键执行纪律（每次会话必读）

**记录是第一要务，30秒内必须完成**

| 规则 | 执行标准 |
|:---|:---|
| **用户输入** | 立即写入 `memory/YYYY-MM-DD.md`，标注时间+T级分类 |
| **AI回复** | 思考过程+最终回复，一并记录，不省略 |
| **双重归档** | memory主文件 + T0-T4分类文件，同步更新 |
| **30秒纪律** | 收到消息→30秒内完成写入→再回复用户 |
| **自检机制** | 每2小时检查遗漏，发现立即补录并承认 |

**禁止事项**：
- ❌ 口头"已记录"实际未写入
- ❌ 只记用户输入，不记AI回复
- ❌ 只创建分类文件，不更新memory主文件

**违反后果**：连续3次同类错误启动重塑流程

## 📁 T0 核心执行规范

### 记录归档规范

**强制要求**：
- 用户输入 + AI思考 + AI回复 → **全部写入** `memory/YYYY-MM-DD.md`
- 同步归档至对应 T0-T4 分类文件
- 30秒内完成，先写文件再回复用户

### 投研记录时段标记

| 时段 | 时间 | 标记 | 内容 |
|------|------|------|------|
| 盘前 | 09:30前 | 【盘前】| 计划制定（需用户"同意"后生效）|
| 盘中 | 09:30-15:00 | 【盘中】| 执行记录、操作、情绪 |
| 盘后 | 15:00后 | 【盘后】| 复盘总结、四问、规则迭代 |

### GitHub备份

- **定时任务**：每日凌晨 1:00 自动 `git add . && git commit && git push`
- **手动备份**：重要变更后立即提交

## 快速记录模板

**核心规则：先记录原文，再分类归档，必须标记时间**

**步骤1：完整记录用户原文**
```markdown
## YYYY-MM-DD HH:MM 原始记录
**用户输入**：（原文完整记录，一字不改）
```

**步骤2：按T级分类处理**

**T1投研内容格式（必须含【盘前/盘中/盘后】）**：
```markdown
## YYYY-MM-DD HH:MM 【盘前/盘中/盘后】 [类型标签]
**用户原文**：（完整引用）
**内容分析**：（AI分析判断）
**来源**：用户直接告知 / 链接文章 / 对话提取
```
